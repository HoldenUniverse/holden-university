#35 - Facility Worlds Phase 3 — Mission & Investigation Engine
What & Why
Transform Facility Worlds from a passive 3D viewer into an active investigation experience. When the user arrives at Perdido, they receive a mission briefing (arrival context + operational issue), hear authentic-sounding radio traffic from the platform, and must investigate to find the problem rather than walking to a pre-marked checkpoint. Equipment is inspectable by looking at it and pressing E — no floating dots, no arcade markers. The scenario panel opens when the user physically reaches and inspects the right piece of equipment.

This is the engine layer that all future Facility Worlds will reuse. Only Perdido is wired up in this task; the data schema and components are designed to be facility-agnostic.

Done looks like
Arrival / Mission briefing

Before Walk Mode starts, a cinematic "arrival card" slides in: a short paragraph setting the scene (helicopter arrival, shift handover, abnormal vibration report, etc.) and a plain-English mission statement ("Investigate the reported corrosion near the production deck riser").
Each of Perdido's 9 hotspots has one authored arrival context + mission statement so the experience varies.
User can dismiss and enter Walk Mode.
Radio traffic

While walking, a small radio-chatter overlay appears periodically in the bottom-left — styled like an offshore comms log (call sign, message, timestamp).
Messages are drawn from a curated pool of 20–30 authentic-sounding offshore comms lines (control room, operations, maintenance supervisor, rope access team) that are plausible for the current mission but never reveal the exact location of the problem.
Radio calls appear every 30–90 s (randomised). No audio required — text only for this phase.
Investigation — no glow, no markers in Walk Mode

Hotspot gold ring markers are hidden while Walk Mode is active. The user must navigate using knowledge of the platform, radio clues, and visual scanning.
When the user looks directly at an interactive piece of equipment (within 18 m, gaze dot-product > 0.92), a subtle white outline renders on that mesh (outline shader or emissive pulse — no floating label).
When within 6 m and looking at a hotspot's anchor zone, the bottom-centre prompt changes to "[E] Inspect Equipment" (same as Phase 2 proximity prompt but now the only visual cue).
Equipment inspection → scenario panel

Pressing E while the prompt is shown opens the HotspotPanel exactly as Phase 2 does (Briefing / Scenario / Documents / D-IMS / CodeIQ / Decision tabs).
Pointer lock releases automatically (already done in Phase 2 fix).
Closing the panel returns to Walk Mode with the "Click to resume" prompt.
Mission state tracking

A lightweight per-session mission state (React context or local state) tracks: mission assigned, hotspots inspected, time elapsed.
After inspecting the mission-target hotspot, a "Finding Logged" toast appears and the arrival card updates to show a brief debrief line ("Corrosion confirmed on riser caisson. Work order raised.").
No consequences/escalation logic in this phase (that is Phase 5).
"Resume Walk Mode" button after panel closes

Panel close button already releases pointer lock. A "Resume Walk Mode" pill button appears in the bottom-centre overlay (same position as the click-to-resume prompt) with a single click to re-engage.
Out of scope
Audio (radio sound effects, PA announcements) — Phase 4.
Ambient life (walking NPCs, crane movement, helicopter, weather) — Phase 4.
Consequence/escalation engine (pressure rise, smoke, alarms) — Phase 5.
Dynamic facility state changes over time — Phase 5.
Any facility world other than Perdido being wired up — future task.
Hotspot anchor recalibration (separate calibration session with user).
