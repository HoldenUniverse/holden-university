Good question — here's concretely how this rig prototype would attach into your actual courses, using the same pattern the platform already uses for other pillars (like the Audiobook player):

1. It becomes a new "pillar" on the Course Hub
Right now each course's hub page shows cards like Flipbook, CodeIQ, Scenarios, Audiobook. This 3D yard would become a new card — something like "Inspection Yard" — on whichever course it's relevant to (most naturally an offshore/mechanical integrity course, or a new "Offshore Operations" course if you want to create one).

2. It ties into enrollment, not the general public
Like the other pillars, only students enrolled in that course would see/access the button. No separate login system — it uses the same account and enrollment they already have.

3. Equipment findings become gradable content
Each clickable piece of equipment (the vessel, pump, exchanger, etc.) already opens an inspection-finding popup. To "attach" this to a course, those findings get tied to real lesson content — e.g., clicking the pressure vessel could show the same corrosion/damage-mechanism info a student is learning in that course's API 510 chapter, and completing an inspection round could log progress the same way a Checkpoint Quiz does today.

4. It shows up on the dashboard/progress the same way
Once wired in, a completed "walk the yard" session would count toward that course's completion percentage, sitting next to Flipbook progress and quiz scores.

Important scope note: what exists right now is still a canvas prototype, not part of the live app — it isn't wired into enrollment, course data, or the dashboard yet. When you're ready to move it from "prototype" to "real feature," that's a distinct build step (what I'd call "graduating" it into the main app) — I'd wire it to a specific course, add the hub card, and connect the equipment findings to real inspection content for that course.
